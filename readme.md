📘 AI 모델을 위한 오목 마스터 v2 기술 설계 및 구현 가이드
이 가이드는 전문적인 수준의 HTML5/JavaScript 기반 오목 게임인 **'오목 마스터: AI v2'**의 전체 아키텍처와 상세 기능을 정의한다. 아래의 기술적 명세와 제약 조건을 모두 충족하는 단일 HTML 파일 코드를 생성하라.

1. 시스템 아키텍처 및 UI 설계 (Visual & UX)
반응형 웹 디자인: 모바일(iOS/Android)과 데스크톱 모두에서 최적의 경험을 제공해야 한다. viewport-fit=cover를 포함하여 노치 디자인을 대응하고, touch-action: none으로 의도치 않은 줌이나 스크롤을 방지한다.

테마 시스템:

기본 테마: 따뜻한 목재 톤(#e9d6a7)의 배경과 고급스러운 다크 그레이 컨트롤러.

불가능 테마: 사용자가 '불가능' 난이도를 선택하면 메인 컬러를 Deep Purple(#8e44ad)로 변경하고 시각적 압박감을 주는 애니메이션을 적용한다.

컴포넌트 구조:

Info-Bar: 좌우에 각 플레이어의 타이머(20초 제한 또는 무제한), 중앙에 상태 메시지 바(Status Bar) 배치.

Canvas: 15x15 그리드. 고해상도 레티나 디스플레이를 대응한 스케일링 처리가 되어야 하며, 바둑알에는 입체감을 주는 Radial Gradient와 그림자 효과를 적용한다.

Control Panel: 무르기, 기권, 메인화면 버튼 배치.

2. 핵심 게임 엔진 (Core Engine & Renju Rules)
렌주룰(Renju Rule)의 엄격한 구현:

흑(선수) 금수: 3-3(두 개 이상의 열린 3 형성), 4-4(두 개 이상의 4 형성), 장륙(6목 이상 나열) 착수를 엄격히 금지한다.

금수 판정 로직: 착수 전 가상으로 돌을 놓아보고 모든 방향(가로, 세로, 대각선 2종)을 스캔하여 패턴을 분석해야 한다. '열린 3'의 정의(양 끝이 비어있어 4가 될 수 있는 상태)를 정확히 계산하라.

백(후수) 특권: 백은 4-4, 3-3, 장륙이 모두 허용되며 이를 통해 승리할 수 있다.

승리 판정: 흑은 정확히 5목일 때만 승리하며, 백은 5목 이상(장륙 포함)일 때 승리한다.

3. 고성능 AI 알고리즘 (Heuristic & Minimax)
평가 함수(Evaluation Function): 각 위치의 가치를 계산할 때 '공격 포인트'와 '수비 포인트'를 별도로 계산한다. 5목 확정(100,000점), 열린 4(10,000점), 닫힌 4/열린 3(3,000점) 등 가중치를 부여한다.

알고리즘 단계별 적용:

초보/중수: 가중치 기반 휴리스틱 탐색. 약간의 무작위성(Randomness)을 부여하여 인간적인 실수를 유도한다.

고수(Hard): Minimax 알고리즘(Depth 1~2) 적용.

불가능(Impossible): Alpha-Beta Pruning이 적용된 Minimax 알고리즘(Depth 2~4). 모든 가능한 수를 탐색하며 사용자의 금수를 유도하는 전략을 우선시한다.

4. 상태 메시지 및 우선순위 시스템 (Smart Status Bar)
메시지 스택 관리:

우선순위 메시지(3초): 금수 알림("⚠️ 33 금수 위치입니다!"), 무르기 알림("무르기 완료 (n/3)"), 시스템 경고.

일반 메시지: 현재 차례("흑 차례"), AI 생각 중...

업데이트 로직: 우선순위 메시지가 출력된 지 3초가 지나면 기본 상태로 복귀한다. 단, 3초 이내에 사용자가 돌을 놓거나 턴이 넘어가면 3초를 기다리지 않고 즉시 새로운 상태로 갱신하여 즉각적인 피드백을 제공한다.

5. 게임 흐름 제어 (Control & Constraints)
무르기(Undo) 제약:

판당 최대 3회로 제한하며 사용 시 "남은 횟수"를 명시한 확인 창을 띄운다.

불가능 난이도에서는 무르기 버튼을 비활성화(disabled)하고 텍스트를 "무르기 불가"로 변경하여 엄격함을 유지한다.

시간 제한: 불가능 난이도에서 흑/백 모두에게 20초 카운트다운을 적용한다. 5초 남았을 때 타이머를 붉은색으로 점멸시키며 카운트 종료 시 즉시 시간패 처리한다.

기권 확인: 기권 버튼 클릭 시 의사를 재확인하는 모달을 띄운다.

6. 데이터 관리 및 복기 (Storage & Replay)
Local History: localStorage를 사용하여 최근 5경기를 저장한다. (날짜, 승자, 총 수수, 착수 경로 리스트, 종료 사유).

복기 모드: 게임 종료 후 '복기 하기'를 누르면 저장된 착수 경로를 캔버스에 수 번호(1, 2, 3...)와 함께 렌더링한다.

7. 기술적 구현 요구사항
Code Quality: 전역 변수를 최소화하고 GameManager, AIEngine, UIManager 등 객체 지향적 또는 모듈형으로 구조화하라.

Performance: AI 연산 시 메인 스레드 점유를 최소화하기 위해 setTimeout 등을 활용해 렌더링 루프가 끊기지 않게 하라.

Library: 외부 라이브러리는 시각 효과를 위한 canvas-confetti 외에는 사용하지 않으며 순수 바닐라 JS로 작성하라.

이상의 지침을 기반으로 오목 게임을 생성하라. 코드는 완성된 형태여야 하며, 사용자가 즉시 실행 가능한 HTML 파일 전체를 출력해야 한다.
